EECU Budget Calculator

Project Description:

This budget calculator was designed to fit EECU's design choices, and act as an online resource for anyone who needs help tracking their personal spending. It utilizes preset income levels based on career choices and set tax values. The input sections utilize JavaScript to create more sub-categories, and CSS to ensure the styling and layout is consistent.

Demonstrated Skills:

  • Development Languages: HTML, CSS, JavaScript
  • Tools: Visual Studio Code, GitHub, Figma
  • Hosting: Netlify @ https://eecu-budget-calculator-netizen.netlify.app/

Concepts Learned:

  • Template Element: I learned how to use HTML template elements alongside JavaScript functions to create multiple “pages” within one HTML file.
  • JavaScript Classes: I grew my understanding in JavaScript through the introduction of adding and removing classes from elements to change their CSS properties.
  • JavaScript Charts: I deepened my knowledge for charts in JavaScript by using online resources and examples to experiment with.

Challenges & Solutions:

  • The Challenge: It was difficult to find a way to make the website on one page, and not need to use localStorage to save data between pages.
  • The Solution: One developer suggested the use of templates, and cycling through premade layouts to give the feel of changing pages without leaving the HTML page.
  • The Challenge: The progress bar was not updating like I wanted it to.
  • The Solution: After researching how classes work in JavaScript, I learned how to add and remove classes to make the progress bar visually update.
  • The Challenge: I struggled to create a chart with live updates to visualize spending categories.
  • The Solution: Studying Chart.js, and learning through trial and error, led me to the final design in my final website, which does have live updates.
  • The Challenge: Even after multiple failed attempts, I could not figure out how to add custom costs for the user.
  • The Solution: I finally came to the conclusion to use the same logic that created new options earlier in the code to create complete elements with the click of a button.

Habits of Mind in Action:

  • Persist: Throughout this project I constantly made mistakes, but due to my persistence, and kept going until I came to my final design.
  • Investigate: To help problem solve, I used multiple websites including W3Schools and Stack Overflow to help problem solve and learn more.
